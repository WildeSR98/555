=========================================================
      PRODUCTION MANAGER - РУКОВОДСТВО И ДОКУМЕНТАЦИЯ      
=========================================================

Приложение "Production Manager" предназначено для автоматизации управления процессом сборки оборудования, выдачи серийных номеров (SN), отслеживания статусов каждого устройства и контроля за выполнением рабочих операций сотрудниками на местах (стендах).

Является десктопным Windows-клиентом (PyQt6) для системы office-task-manager.

---------------------------------------------------------
1. ТЕХНОЛОГИЧЕСКИЙ СТЕК
---------------------------------------------------------
* Язык: Python 3
* Интерфейс: PyQt6 (Темная тема, векторная графика, автоподгонка колонок QTableWidget).
* База данных: SQLAlchemy (поддерживает SQLite, готов к мгновенной бесшовной миграции на PostgreSQL локально или удаленно).
* Графики и аналитика: matplotlib
* Инструмент сборки: PyInstaller

---------------------------------------------------------
2. РОЛИ ПОЛЬЗОВАТЕЛЕЙ И АВТОРИЗАЦИЯ
---------------------------------------------------------
В системе предусмотрено жесткое разграничение прав доступа. Авторизация происходит по уникальному пин-коду.
* АДМИНИСТРАТОР (Admin): Имеет доступ ко всем вкладкам, может создавать новые проекты, управлять справочниками, удалять операции и сбрасывать пулы серийных номеров.
* СБОРЩИК/ТЕСТИРОВЩИК (Worker): Имеет доступ только к интерфейсу "Сканирование". Не видит глобальных дашбордов и не может модифицировать базу настроек.

---------------------------------------------------------
3. СТРУКТУРА ПРИЛОЖЕНИЯ (ВКЛАДКИ)
---------------------------------------------------------

[ Вкладка "Пул SN" ]
Независимый менеджер истории серийных номеров. Оборудование разделено на базовые "Категории", имеющие свой уникальный префикс (например, 60LXTR...).
* Внутри категорий создаются "модели". Для каждой заводится свой собственный, независимый счетчик.
* Ручной сдвиг (Якоря): возможность перепрыгнуть количество номеров. Система создаст якорную пустую запись, от которой продолжится генерация.
* Используется многопоточное безопасное резервирование записей для исключения дубликатов SN.

[ Вкладка "Проекты" ]
Используется для планирования работы производственной линии.
* Администратор создает "Проект": название, внутренний код, менеджер, добавляет оборудование из Пула SN.
* При сохранении проекта система автоматически генерирует нужное число устройств, назначает им свободные серийные номера.
* Проекты можно удалять. В этом случае устройства аннулируются, но их номера в "Пуле" остаются в истории как использованные.

[ Вкладка "Конвейер" ]
Отображает полную цепочку этапов прохождения устройства (Сборка -> ОС -> Тестирование -> ОТК -> Упаковка). Эти данные регламентируют, какой статус будет получать оборудование при сканировании.

[ Вкладка "Сканирование" ]
Основной интерфейс для сотрудников производственной линии (сборщиков).
1) Работник выбирает свой стенд.
2) Сканирует серийный номер с устройства (штрихкод-сканером).
3) Устройство находится в базе, записывается лог (дата, стенд, работник).
4) Статус устройства продвигается на следующий шаг (по правилам Конвейера).
5) Курсор моментально фокусируется обратно для сканирования следующего.

[ Вкладка "Статус SN" ]
Поисковик для отслеживания устройства. Показывает текущий статус, лог перемещений, партномер и проект.

[ Вкладки "Дашборд" и "Аналитика" ]
Сводка об общем количестве устройств во всех системах и проектах. Визуализация статистики (графики matplotlib) по загруженности линии и прогрессу сборки.

---------------------------------------------------------
4. УСТАНОВКА И ЗАПУСК
---------------------------------------------------------

1. Установите зависимости:
   > pip install -r requirements.txt

2. Настройте файл .env (скопируйте из .env.example):
   # Пример для SQLite
   DB_TYPE=sqlite
   DB_PATH=db.sqlite3

   # Пример для PostgreSQL
   DB_TYPE=postgresql
   DB_HOST=your-server.com
   DB_PORT=5432
   DB_NAME=production_db
   DB_USER=app_user
   DB_PASSWORD=your_password

3. Команда для запуска:
   > python src/main.py

4. Команда для сборки .exe файла:
   > python build.py
   (Итоговый файл будет в каталоге dist/ProductionManager.exe)
