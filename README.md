# Production Manager 🏭

**Production Manager** — это десктопное Windows-приложение, созданное для автоматизации управления процессом сборки оборудования, выдачи серийных номеров (SN), отслеживания статусов каждого устройства и контроля за выполнением рабочих операций сотрудниками на местах (стендах).

Приложение является клиентом для системы `appwin`.

## 🛠 Технологический стек

- **Язык**: Python 3
- **Графический интерфейс (GUI)**: PyQt6 (Темная тема, векторная графика, автоподгонка таблиц)
- **ORM**: SQLAlchemy
- **База данных**: Поддержка SQLite (по умолчанию для разработки) и PostgreSQL (рекомендуется для продакшена). Готово к мгновенной бесшовной миграции.
- **Визуализация данных (Графики)**: matplotlib
- **Сборка приложения**: PyInstaller

## 👥 Роли пользователей и Авторизация

В системе предусмотрено жесткое разграничение прав доступа посредством уникальных пин-кодов сотрудников:

1. **АДМИНИСТРАТОР (Admin)**
   - Имеет полный доступ ко всем вкладкам системы.
   - Может создавать новые проекты, управлять справочниками.
   - Имеет право удалять операции и сбрасывать пулы серийных номеров.
2. **СБОРЩИК / ТЕСТИРОВЩИК (Worker)**
   - Ограниченный доступ: видит только интерфейс "Сканирование".
   - Не имеет доступа к глобальным дашбордам, не может модифицировать структуру базы или настройки проектов.

## 📑 Основные разделы (Вкладки)

### 1. Пул SN (Генерация серийных номеров)

Независимый менеджер истории серийных номеров. Оборудование разделено на базовые "Категории" (например: Tioga, JBOX, Мониторы, Стойки).
Каждая категория имеет уникальный префикс (например, `60LXTR...` для Tioga).

- Внутри категорий можно создавать уникальные **"модели"** (например, `JBOX Lite`).
- Для каждой модели заводится собственный независимый счетчик.
- **Ручной сдвиг (Якоря)**: возможность "перепрыгнуть" пул номеров. При нажатии "Задать счетчик" создается якорная запись-пустышка, от которой продолжится генерация номеров, что позволяет синхронизироваться с реальным производством.
- Исключена вероятность дубликатов SN благодаря многопоточному безопасному резервированию записей.

### 2. Проекты

Раздел для планирования работы производственной линии.

- Администратор задает название, внутренний код и ответственного менеджера.
- Добавляется оборудование (можно использовать несколько рядов устройств, выбирая модели из "Пула SN").
- Для каждой позиции указывается партномер и количество.
- При сохранении система генерирует нужное число устройств, забирая следующие свободные серийные номера и помечая их как "Занят".
- При удалении проекта устройства аннулируются, но использованные серийные номера остаются в истории, чтобы не нарушать счетчики.

### 3. Конвейер

Справочная визуализация полной цепочки этапов для устройства (например: *Сборка → ОС → Тестирование → ОТК → Упаковка*). Эти данные регламентируют маршрут движения оборудования и изменение его статуса при сканировании.

### 4. Сканирование

Основной рабочий интерфейс для производственной линии.

1. Сотрудник выбирает свой стенд.
2. Фокус ввода устанавливается в специальное поле, сотрудник сканирует SN устройства (через штрихкод-сканер).
3. Система находит устройство по SN и записывает в лог перемещений: дата-штамп, выбранный стенд, пин-код работника.
4. Статус устройства автоматически продвигается на следующий производственный шаг в соответствии с "Конвейером".
5. Курсор моментально возвращается на поле ввода, позволяя сканировать следующее устройство без использования мыши.

### 5. Статус SN

Универсальный поисковик для оператора или администратора. Позволяет вручную найти серийный номер и посмотреть:

- Текущий статус устройства (Сборка, Тесты и др.).
- Историю (лог) перемещений между стендами.
- Привязанный Партномер и наименование Проекта.

### 6. Дашборд и Аналитика

- **Дашборд**: Сводная информация о количестве устройств во всех системах и проектах, быстрый взгляд на занятость производственной линии и текущие активные сессии.
- **Аналитика**: Встроенные наглядные графики (на базе `matplotlib`), демонстрирующие статистику сборки проектов и уровень загруженности стендов.

## 💻 Установка и запуск

### 1. Установка зависимостей

Убедитесь, что у вас установлен Python 3. Затем выполните команду в корневой папке проекта:

```bash
pip install -r requirements.txt
```

### 2. Настройка базы данных

Скопируйте файл `.env.example` в `.env` и настройте параметры под вашу среду.

**Для разработки (SQLite)**:

```ini
DB_TYPE=sqlite
DB_PATH=db.sqlite3
```

**Для продакшена (PostgreSQL)**:
В рабочей среде рекомендуется использовать PostgreSQL. В проекте предусмотрен защищенный механизм миграции.

```ini
DB_TYPE=postgresql
DB_HOST=your-server.com
DB_PORT=5432
DB_NAME=production_db
DB_USER=app_user
DB_PASSWORD=your_password
```

### 3. Запуск приложения в режиме разработчика

```bash
python src/main.py
```

### 4. Сборка исполняемого файла (.exe)

Для компиляции проекта в standalone-приложение выполните скрипт сборки:

```bash
python build.py
```

Готовый к распространению файл можно будет найти по пути: `dist/ProductionManager.exe`
