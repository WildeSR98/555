## [Unreleased] — 2026-04-20

### Добавлено — SN Pool: управление категориями устройств

- Новая таблица `pm_device_category` (code PK, display_name, sn_prefix, sort_order)
- Миграция `migrate_device_categories.py`: сидирование из `Device.DEVICE_TYPE_DISPLAY`
- API: `POST /api/sn-pool/categories`, `PATCH /api/sn-pool/categories/{code}`
- UI: кнопка «📂 Добавить категорию» в шапке, кнопка «⚙️» при hover на категорию
- Модалка создания: код (автозаглавные), название, префикс SN
- Дерево SN Pool теперь читает категории из БД вместо хардкода

### Добавлено — SN Pool: переименование моделей устройств

- `PATCH /api/sn-pool/models/{model_id}` — переименование с проверкой дубликатов
- Кнопка «✏️» появляется при hover на модели в дереве sidebar
- Заголовок таблицы обновляется при переименовании активной модели

### Добавлено — Маршрутные листы: вкладка «Проекты»

- Новая модель `ProjectRouteStage` (таблица `pm_project_route_stage`)
- Новый API модуль `project_routes_api.py`:
  - `GET /api/project-routes/projects` — проекты с устройствами
  - `GET /api/project-routes/{pid}/device/{type}` — этапы (override или global fallback)
  - `PUT /api/project-routes/{pid}/device/{type}` — сохранить индивидуальный маршрут
  - `DELETE /api/project-routes/{pid}/device/{type}` — сбросить до глобального
- Вкладки «📋 Маршруты» / «📁 Проекты» на странице `/route-configs`
- Аккордеон проектов: раскрытие → типы устройств → редактор этапов с тоглами
- Изоляция маршрутов: изменения проекта А не влияют на проект Б
- Fallback: без override используется глобальный `RouteConfig` для типа
- Авто-переключение на вкладку «Проекты» при GET-параметре `?tab=projects`

### Добавлено — Проекты: кнопка «Маршрутные листы»

- Кнопка «📋 Маршрутные листы» появляется в деталях проекта при его выборе
- Перенаправляет на `/route-configs?tab=projects` с авто-открытием вкладки

### Добавлено — CHANGELOG.md

- Создан файл истории изменений проекта по стандарту Keep a Changelog

## [v5.x] — 2026-04-17

### Добавлено — Drag-and-Drop в конфигурации маршрутов

- Перетаскивание этапов конвейера в редакторе маршрутных листов (HTML5 DnD)
- Сохранение `order_index` через `saveConfig()`
- Визуальный feedback: прозрачность + пунктирная рамка при перетаскивании

### Добавлено — Кастомные этапы в маршрутах

- Кнопка «Добавить этап» → prompt() с названием
- Хранение как `CUSTOM::Название` без миграции БД
- Синий левый бордер, тег «польз. этап», кнопка «✕» для удаления
- Перетаскиваются наравне с системными этапами

### Добавлено — Разбивка маршрутов при создании проекта

- При создании проекта с несколькими типами устройств показывает разбивку маршрутов
- Умный автовыбор маршрута при множестве совпадений для одного типа

### Добавлено — WebSocket обновления в реальном времени

- Вкладка «Проекты» обновляется автоматически без ручного рефреша
- Toast-уведомления о новых событиях
- Живые обновления дашборда и пайплайна

## [v5.0] — 2026-04-15

### Добавлено — Запуск веб-версии

- Полный веб-интерфейс FastAPI (миграция с Qt-only)
- PostgreSQL как основная БД (Docker)
- Система маршрутных листов (`RouteConfig`, `RouteConfigStage`)
- Новые этапы конвейера: Warehouse, Accounting, Packing
- Аутентификация через сессии FastAPI
- Архив проектов
- Аналитика и дашборд
