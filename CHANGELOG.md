## [Unreleased] — 2026-04-20

### ✨ Добавлено

#### 🗺️ Маршрутные листы по проектам (`ProjectRouteStage`)
- Новая таблица `pm_project_route_stage` — индивидуальные маршруты на уровне проекта
- Каждый проект + тип устройства может иметь собственный маршрут, изолированный от других
- Fallback: если override не задан — используется глобальный `RouteConfig` для данного типа
- Новый API модуль `web/api/project_routes_api.py`:
  - `GET /api/project-routes/projects` — список проектов с устройствами
  - `GET /api/project-routes/{pid}/device/{type}` — этапы (override или global)
  - `PUT /api/project-routes/{pid}/device/{type}` — сохранить override
  - `DELETE /api/project-routes/{pid}/device/{type}` — сбросить до глобального

#### 📋 Вкладка «Проекты» на странице маршрутных листов
- Добавлены таб-кнопки «Маршруты» / «Проекты» на странице `/route-configs`
- Аккордеон проектов в сайдбаре → типы устройств внутри
- Правая панель: список этапов с переключателями (enable/disable) per-project
- Кнопки: «💾 Сохранить для проекта» и «↺ Сбросить до глобального»
- Бейдж «изменён» у типов с индивидуальным маршрутом
- Авто-переключение на вкладку «Проекты» при переходе с URL-параметром `?tab=projects`

#### 🔗 Кнопка «Маршрутные листы» в деталях проекта
- На странице «Проекты» при клике на проект появляется кнопка «📋 Маршрутные листы»
- Перенаправляет на `/route-configs?tab=projects` с авто-открытием вкладки

#### 📂 Управление категориями устройств (SN Pool)
- Новая таблица `pm_device_category` — динамические категории (overlay хардкода)
- Миграция `migrate_device_categories.py`: сидируется из `Device.DEVICE_TYPE_DISPLAY`
- API: `GET/POST /api/sn-pool/categories`, `PATCH /api/sn-pool/categories/{code}`
- UI: кнопка «📂 Добавить категорию», кнопка «⚙️» при наведении на категорию
- Модалка создания: код (автозаглавные), название, префикс SN
- Модалка редактирования: название + префикс (код не меняется)
- Дерево SN теперь использует БД-категории вместо хардкода

#### ✏️ Переименование моделей устройств (SN Pool)
- Новый API: `PATCH /api/sn-pool/models/{model_id}` — переименование модели
- Кнопка «✏️» появляется при наведении на модель в дереве
- Изменение: prompt() с текущим именем → PATCH → дерево обновляется
- Если переименована активная модель — заголовок таблицы обновляется сразу

## [v5.x] — 2026-04-17

### ✨ Добавлено

#### 🔄 Drag-and-Drop в конфигурации маршрутов
- Перетаскивание этапов конвейера в редакторе маршрутных листов (HTML5 DnD)
- Сохранение `order_index` через `saveConfig()`
- Визуальный feedback: прозрачность + пунктирная рамка при перетаскивании

#### ➕ Кастомные этапы в маршрутах
- Кнопка «Добавить этап» → prompt() с названием
- Хранение как `CUSTOM::Название` без миграции БД
- Синий левый бордер, тег «польз. этап», кнопка «✕» для удаления
- Перетаскиваются наравне с системными этапами

#### 📋 Разбивка маршрутов в создании проекта
- При создании проекта с несколькими типами устройств — показывает разбивку
  маршрутов по типам внизу модального окна
- Умный автовыбор маршрута при множестве совпадений для одного типа

#### 🌐 WebSocket обновления в реальном времени
- Таб «Проекты» обновляется автоматически без ручного обновления страницы
- Toast-уведомления о новых событиях
- Живые обновления дашборда и пайплайна
