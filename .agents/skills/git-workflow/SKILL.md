# Git Workflow

При любой разработке или внесении изменений в код проекта необходимо строго соблюдать следующий Git-процесс:

1. **Pull (Синхронизация)**: Перед началом работы всегда выполнять `git pull` в основной ветке, чтобы подтянуть актуальные изменения с сервера.
2. **Ветвление**: Каждая новая фича или изменение должно выполняться в отдельной ветке. Создавайте ветку с помощью `git checkout -b <имя_ветки>`. Имя ветки должно отражать суть задачи (например: `feature/login-page`, `bugfix/table-update`).
3. **Разработка и Коммиты**: Выполняйте код ("пилите фичу"). Каждое изменение должно сопровождаться осмысленными коммитами с подробным описанием `git commit -m "..."`, чтобы формировалась понятная история проекта.
4. **Push (Публикация)**: После окончания работы отправляйте ветку в удаленный репозиторий: `git push -u origin <имя_ветки>`.

Этот процесс гарантирует наличие чистой истории, независимость фич и безопасность основной кодовой базы проекта.
