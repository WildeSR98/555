=========================================================
      PRODUCTION MANAGER - РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ      
=========================================================

Приложение "Production Manager" предназначено для автоматизации управления процессом сборки оборудования, выдачи серийных номеров, отслеживания статусов каждого устройства и контроля за выполнением рабочих операций сотрудниками на местах (стендах).

---------------------------------------------------------
1. Роли пользователей и Авторизация
---------------------------------------------------------
В системе предусмотрено жесткое разграничение прав доступа. Каждому сотруднику выдается уникальный пин-код.
* АДМИНИСТРАТОР (Admin): Имеет доступ ко всем вкладкам, может создавать новые проекты, управлять справочниками, удалять операции и сбрасывать пулы серийных номеров.
* СБОРЩИК/ТЕСТИРОВЩИК (Worker): Имеет доступ только к интерфейсу "Сканирование". Не видит глобальных дашбордов и не может случайным образом модифицировать базу.

---------------------------------------------------------
2. Вкладка "Пул SN" (Генерация セрийных номеров)
---------------------------------------------------------
Эта вкладка - независимый менеджер истории серийных номеров. Оборудование разделено на базовые "Категории" (Tioga, JBOX, Мониторы, Стойка и т.д.), каждая из которых имеет свой уникальный префикс (например, `60LXTR...` для Tioga).
* Внутри категорий администратор может создавать свои уникальные "модели" (Например, в категории JBOX можно создать `JBOX Lite`). 
* Для каждой модели заводится свой собственный, полностью независимый счетчик, поддерживающий возможность задать свой уникальный префикс.
* Ручной сдвиг (Якоря): Если необходимо "перепрыгнуть" определенное количество номеров (или соединить программу с реальным производством), над таблицей модели нажимается кнопка "Задать счетчик". Система создаст желтую искусственную якорную запись-пустышку, от которой продолжится генерация серийных номеров для новых проектов.

---------------------------------------------------------
3. Вкладка "Проекты"
---------------------------------------------------------
Служит для планирования работы производственной линии. Администратор создает "Проект":
* Указывает название, внутренний код (опционально) и менеджера.
* Добавляет оборудование. Вы можете добавить несколько рядов устройств, выбирая созданные в "Пуле SN" модели устройств из выпадающего списка. Для каждого ряда указывается партномер и требуемое количество.
* При сохранении проекта, система автоматически: генерирует нужное число устройств, физически закрепляет за ними следующий по порядку свободный серийный номер из пула выбранной модели и помечает его как "Занят".
Проекты можно удалять. В случае удаления проекта, все привязанные устройства также аннулируются, однако их серийные номера в "Пуле" остаются в истории как использованные, не нарушая последовательность счетчиков.

---------------------------------------------------------
4. Вкладка "Конвейер"
---------------------------------------------------------
Здесь отображается полная цепочка этапов, которые должно пройти устройство (например: Сборка, ОС, Тестирование, ОТК, Упаковка). Это справочная настройка - ее данные регламентируют, какой статус будет получать оборудование при сканировании.

---------------------------------------------------------
5. Вкладка "Сканирование"
---------------------------------------------------------
Основной интерфейс для сотрудников производственной линии (сборщиков).
1) Работник выбирает свой стенд (рабочее место).
2) Встает в поле ввода и "сканирует" серийный номер с устройства (штрихкод-сканером, который работает как клавиатура).
3) Система тут же находит этот номер в базе. Если устройство существует, в лог записывается дата-штамп, выбранный стенд, пин-код работника.
4) Статус самого устройства продвигается на следующий производственный шаг в соответствии с маршрутом из вкладки "Конвейер".
После завершения процесса сканирования курсор моментально возвращается (фокусируется обратно) на поле ввода серийника, чтобы сборщик мог сканировать следующее устройство в партии без необходимости прикасаться к мышке.

---------------------------------------------------------
6. Вкладка "Статус SN"
---------------------------------------------------------
Универсальный поисковик. Позволяет оператору или администратору вручную пробить серийный номер, чтобы узнать текущий статус привязанного устройства (Сборка, Тесты и др), его лог перемещений между стендами, а также посмотреть привязанный Партномер и наименование Проекта.

---------------------------------------------------------
7. Вкладка "Дашборд" и "Аналитика"
---------------------------------------------------------
* Дашборд: Выводит сводку об общем количестве устройств во всех системах и проектах, быстрый взгляд на занятость линии.
* Аналитика: Встроенные графики (построенные через Matplotlib), наглядно демонстрирующие статистику сборки проектов (сколько собрано, сколько в работе), или загруженность стендов.

---------------------------------------------------------
Технические особенности
---------------------------------------------------------
Язык: Python 3.
Интерфейс: PyQt6 (Темная тема, векторная графика).
База данных: SQLAlchemy (поддерживает SQLite, готов к мгновенной бесшовной миграции на PostgreSQL при необходимости).
Отслеживаются как кастомные префиксы, так и логика многопоточного безопасного резервирования записей счетчиков для исключения дубликатов SN.
