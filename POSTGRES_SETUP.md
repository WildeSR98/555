## 1️⃣ Установка PostgreSQL

### Вариант А: Windows

1. Скачайте установщик с [официального сайта](https://www.postgresql.org/download/windows/).
2. Запустите установку:
   - Выберите версию (рекомендуется последняя стабильная).
   - Укажите пароль для суперпользователя `postgres` (**запомните его!**).
   - Порт оставьте `5432`.
   - Локаль: `Russian_Russia.UTF-8` (или английская, если нужна стандартная сортировка).
3. Снимите галочку "Stack Builder" в конце установки.

### Вариант Б: Linux (Ubuntu/Debian)

```bash
sudo apt update
sudo apt install postgresql postgresql-contrib
sudo systemctl enable postgresql
sudo systemctl start postgresql
```

Проверка статуса:

```bash
sudo systemctl status postgresql
```

## 2️⃣ Настройка сетевого доступа

По умолчанию PostgreSQL принимает подключения только с localhost. Для сетевого доступа нужно изменить конфигурацию.

### Шаг 1: Изменение `postgresql.conf`

Найдите файл конфигурации:

- **Windows:** `C:\Program Files\PostgreSQL\16\data\postgresql.conf`
- **Linux:** `/etc/postgresql/16/main/postgresql.conf`

Найдите строку `listen_addresses` и измените её:

```ini
listen_addresses = '*'
```
> Это позволит принимать подключения со всех сетевых интерфейсов.

### Шаг 2: Изменение `pg_hba.conf`

Файл находится в той же директории, что и `postgresql.conf`.

Добавьте в конец файла правило для вашей подсети (замените `192.168.1.0/24` на вашу сеть):

```text
# TYPE  DATABASE        USER            ADDRESS                 METHOD
host    production_db   prod_user       192.168.1.0/24          scram-sha-256
```

> **Важно:** Метод `scram-sha-256` используется в современных версиях PostgreSQL. Если у вас старая версия (до 10), используйте `md5`.

### Шаг 3: Перезапуск службы

**Windows (PowerShell от администратора):**

```powershell
Restart-Service -Name postgresql-x64-16
```

**Linux:**

```bash
sudo systemctl restart postgresql
```

### Шаг 4: Настройка брандмауэра

Откройте порт **5432** для входящих подключений:

**Windows:**

```powershell
New-NetFirewallRule -DisplayName "PostgreSQL" -Direction Inbound -LocalPort 5432 -Protocol TCP -Action Allow
```

**Linux (UFW):**

```bash
sudo ufw allow 5432/tcp
```

## 3️⃣ Создание базы данных и пользователя

Войдите в консоль PostgreSQL от имени суперпользователя.

**Windows:**

```powershell
cd "C:\Program Files\PostgreSQL\16\bin"
.\psql.exe -U postgres
```

**Linux:**

```bash
sudo -u postgres psql
```

Выполните следующие SQL-команды:

```sql
-- 1. Создаем пользователя для приложения
CREATE USER prod_user WITH PASSWORD 'strong_password_here';

-- 2. Создаем базу данных с владельцем prod_user
CREATE DATABASE production_db OWNER prod_user;

-- 3. Предоставляем полные права на базу данных
GRANT ALL PRIVILEGES ON DATABASE production_db TO prod_user;

-- 4. Важно: даем права на схему public (для PostgreSQL 15+)
\c production_db
GRANT ALL ON SCHEMA public TO prod_user;

-- Выход
\q
```

> 🔒 **Безопасность:** Замените `strong_password_here` на сложный пароль!

## 4️⃣ Подготовка приложения

### Шаг 1: Обновление зависимостей

Убедитесь, что установлен драйвер PostgreSQL:

```bash
pip install psycopg2-binary
```

Или обновите все зависимости из `requirements.txt`:

```bash
pip install -r requirements.txt
```

### Шаг 2: Настройка файла `.env`

Создайте или отредактируйте файл `.env` в корне проекта:

```ini
# Тип базы данных
DB_TYPE=postgresql

# Параметры подключения к PostgreSQL
DB_HOST=192.168.1.100       # IP-адрес сервера PostgreSQL
DB_PORT=5432                # Порт (по умолчанию 5432)
DB_NAME=production_db       # Имя базы данных
DB_USER=prod_user           # Имя пользователя
DB_PASSWORD=strong_password_here  # Пароль пользователя

# Путь к исходной SQLite базе (для миграции)
SQLITE_PATH=production.db
```

> ⚠️ **Важно:** Убедитесь, что `DB_HOST` содержит правильный IP-адрес сервера PostgreSQL в вашей сети.

## 5️⃣ Миграция данных

Скрипт `migrate_sqlite_to_pg.py` автоматически перенесет все данные из локальной SQLite базы в PostgreSQL.

### Запуск миграции

```bash
python migrate_sqlite_to_pg.py
```

### Что делает скрипт

1. ✅ Проверяет наличие файла SQLite (`production.db`).
2. ✅ Подключается к PostgreSQL по параметрам из `.env`.
3. ✅ Читает структуру и данные из всех таблиц SQLite.
4. ✅ Создает соответствующие таблицы в PostgreSQL (если они еще не созданы приложением).
5. ✅ Переносит все записи, обрабатывая типы данных.
6. ✅ Фиксирует транзакцию или выполняет откат при ошибке.

### Пример вывода

```text
🔍 Проверка исходной базы SQLite: production.db...
✅ Найдено таблиц: 8
🔍 Подключение к PostgreSQL (192.168.1.100/production_db)...
✅ Подключение успешно
🚀 Начало миграции...

📦 Миграция таблицы: users...
   ✅ Перенесено строк: 5/5
📦 Миграция таблицы: projects...
   ✅ Перенесено строк: 12/12
...
🎉 Миграция завершена успешно!
📊 Всего перенесено записей: 347
```

> ⚠️ **Внимание:** Перед миграцией сделайте резервную копию SQLite базы:
> ```bash
> copy production.db production.db.backup
> ```

```bash
python src/main.py
```

Или запустите скомпилированную версию:

```bash
./dist/ProductionManager/ProductionManager.exe
```

### Шаг 2: Проверка подключения

1. Войдите в приложение под учетной записью администратора.
2. Откройте любую вкладку (Проекты, Сотрудники и т.д.).
3. Убедитесь, что данные отображаются корректно.
4. Попробуйте создать новую запись — она должна сохраниться в PostgreSQL.

3. Убедитесь, что видны те же данные, что и на основном компьютере.

## 8️⃣ Решение проблем

### ❌ Ошибка: "could not connect to server: Connection refused"

**Причина:** PostgreSQL не слушает сетевые подключения или порт закрыт брандмауэром.

**Решение:**

1. Проверьте `listen_addresses = '*'` в `postgresql.conf`.
2. Убедитесь, что служба PostgreSQL перезапущена.
3. Проверьте правила брандмауэра (порт 5432 должен быть открыт).
4. Проверьте, что IP-адрес в `.env` совпадает с реальным IP сервера.

### ❌ Ошибка: "no pg_hba.conf entry for host..."

**Причина:** В файле `pg_hba.conf` нет правила для вашего IP-адреса.

**Решение:**

Добавьте правило в `pg_hba.conf`:

```text
host    production_db   prod_user   <ВАШ_IP>/32   scram-sha-256
```

Перезапустите PostgreSQL.

### ❌ Ошибка: "password authentication failed"

**Причина:** Неверный пароль или имя пользователя.

**Решение:**

1. Проверьте правильность `DB_USER` и `DB_PASSWORD` в `.env`.
2. Убедитесь, что пользователь создан в PostgreSQL:

   ```sql
   \du
   ```

### ❌ Ошибка: "database does not exist"

**Причина:** База данных не создана или указано неверное имя.

**Решение:**

Проверьте список баз данных:

```sql
\l
```

Убедитесь, что `DB_NAME` в `.env` совпадает с именем базы.

### ❌ Ошибка при миграции: "relation already exists"

**Причина:** Таблицы уже были созданы в PostgreSQL (например, при предыдущем запуске приложения).

**Решение:**
Это нормальная ситуация. Скрипт пытается вставить данные в существующие таблицы. Если данные дублируются, очистите целевую базу перед повторной миграцией:

```sql
-- Внимание: это удалит все данные!
DROP SCHEMA public CASCADE;
CREATE SCHEMA public;
GRANT ALL ON SCHEMA public TO prod_user;
```
